## 5 — Email & SMS Alert Subscriptions

**Install:** `npm install resend twilio @types/twilio --workspace=server`

**New Replit Secrets needed:**
```
RESEND_API_KEY          from resend.com (free, no credit card)
RESEND_FROM_EMAIL       use onboarding@resend.dev for dev/testing
TWILIO_ACCOUNT_SID      optional — leave empty to disable SMS
TWILIO_AUTH_TOKEN       optional
TWILIO_PHONE_NUMBER     optional
```

**New Prisma model** — add to schema then run `prisma migrate dev --name add_alert_subscriptions`:
```prisma
model AlertSubscription {
  id                String   @id @default(cuid())
  email             String
  phone             String?
  locationName      String
  lat               Float
  lng               Float
  radiusMiles       Int      @default(25)
  species           String   @default("all")
  isEmailVerified   Boolean  @default(false)
  verificationToken String   @unique @default(cuid())
  unsubscribeToken  String   @unique @default(cuid())
  lastAlertSentAt   DateTime?
  createdAt         DateTime @default(now())
  @@unique([email, lat, lng])
  @@index([lat, lng])
}
```

**New API routes** — create `server/src/routes/alerts.ts` and register at `/api/v1/alerts`:
```
POST /subscribe         Validate email+location, upsert subscription, send verification email via Resend
GET  /verify?token=     Mark isEmailVerified=true → redirect to /alert-confirmed
GET  /unsubscribe?token= Delete subscription → redirect to /unsubscribed
GET  /config            Return { smsEnabled: boolean } based on whether TWILIO vars are set
```

**Alert service** — create `server/src/services/alertService.ts`:
- `sendNewPetAlerts(pet)` — fetch all verified subscribers, filter by Haversine distance ≤ radiusMiles and species match, send email via Resend + SMS via Twilio (if phone + Twilio configured), update `lastAlertSentAt`
- Process in chunks of 20 with `Promise.allSettled` to avoid rate limits
- Wrap everything in try/catch — never let alert failure affect pet creation

**Trigger in pets route** — after `prisma.pet.create()` succeeds, fire and forget:
```typescript
sendNewPetAlerts(newPet).catch(err => logger.error('Alert dispatch failed:', err));
```

**Email content (build as HTML):**
- Alert email: pet photo, name, species, last seen date + address, amber CTA button → pet profile URL, unsubscribe link in footer
- Verification email: confirm button → `/api/v1/alerts/verify?token=`, plain clean layout

**New client component** `AlertSubscribeModal.tsx`:
- Fields: email (required), phone (optional, hide if `smsEnabled` is false), radius segmented control (5/10/25/50 mi), species segmented control (All/Dogs/Cats), read-only location display with "Change" link
- Submit → POST `/api/v1/alerts/subscribe` → show "Check your email to confirm" success state
- Footer: "Free to use. Unsubscribe anytime — no account needed."

**Add subscribe trigger in 3 places:**
1. Home page — dismissable banner between hero and feed: "🔔 Get notified when pets go missing near you. [Set Up Free Alerts →]" — `#EEF2FF` bg, `4px solid #4F46E5` left border
2. Pet profile page — small link below "Add Sighting" button: "🔔 Get alerts for pets lost near you →"
3. After sighting submitted — prompt before redirect: "Want alerts for future pets near you? [Yes] [No thanks]" — auto-redirect after 4s

**New client pages** (add to React Router):
- `/alert-confirmed` — "You're all set! Alerts are active for [location]." + "Browse Lost Pets →"
- `/unsubscribed` — "You've been unsubscribed." + "Resubscribe" and "Go Home"
